# Changelog

## v1.3.0 (2026-05-23)

### Web 安装向导

- 新增 `install.php` 一键部署向导，浏览器访问即可完成安装，无需手动操作数据库和配置文件。
- 自动测试数据库连接、创建数据库、执行 `init.sql` 建表、写入 `api/config.php`。
- 安装完成后生成 `install.lock` 锁定文件，再次访问 `install.php` 仅显示"系统已安装"，拒绝运行。
- 智能检测目标数据库是否已有数据，提供三种处理方式：
  - **保留现有数据**：仅更新配置文件，不触碰数据库。
  - **覆盖安装**：删除所有旧表后重建（需二次确认）。
  - **取消**：返回修改配置。
- `api/config.php` 数据库参数改为占位符（`{{DB_HOST}}` 等），由安装向导自动替换，也支持手动编辑。
- `init.sql` 分类预置数据改用 `INSERT IGNORE`，确保重复执行幂等。

### 过期推算修复

- 修复「有生产日期但无保质期」的物品不显示过期时间的问题。保质期为空的物品统一按存储类型兜底推算（冷藏 +5 天 / 冷冻 +90 天）。

## v1.2.2 (2026-05-23)

### Bug 修复

- **修复「有生产日期但无保质期」的物品不显示过期时间**：过期计算的三分支逻辑漏掉了此场景，导致物品直接穿透所有分支、`expire_date` 为空。现将第三个分支改为 `else` 兜底，只要保质期为空就按存储类型默认推算（冷藏 +5 天 / 冷冻 +90 天）。
- 同步修复 `dashboard.php` 中三处 SQL 的第三组 WHERE 条件（`production_date IS NULL AND` 改为仅判断 `shelf_life_value IS NULL`），确保临期统计和已过期统计覆盖此场景。

## v1.2.1 (2026-05-23)

### 汇总看板增强

- **「未标注物品预警」区域正式启用** — 之前 HTML 占位但从未渲染。现在展示所有无生产日期、无保质期、存放超过 3 天的物品。

  - 冷冻：`未填生产日期和保质期时，默认 添加日期+90天 过期`

## v1.2.0 (2026-05-23)

### 添加物品独立为功能 Tab

- **"添加物品"从库存浏览中分离**，作为独立的导航 Tab（位于汇总看板之后、库存浏览之前）。
- 添加表单直接内嵌在页面中，无需弹窗，支持连续添加。

### 冷藏 / 冷冻区分

- 新增**存储类型**选择（❄️ 冷藏 / 🧊 冷冻），添加物品时选择。
- 物品卡片和编辑弹窗中均展示存储类型。
- **冷冻默认过期推算**：无生产日期且无保质期的冷冻物品，自动按「添加日期 + 90 天」推算过期时间（冷藏仍为 +5 天）。

### 库存浏览分页

- 库存浏览默认展示**全部冰箱的所有物品**，30 条一页，按过期时间排序。
- 底部新增分页控件（首页/上一页/页码/下一页/末页）。
- 每张物品卡片新增冰箱名称显示（🧊）。
- 切换筛选条件自动回到第 1 页。

### 数据库变更

- `items` 表新增 `storage_type ENUM('cold','frozen')` 列。

## v1.1.0 (2026-05-23)

### 统计逻辑优化

- **仪表板"物品总数量"改为 `COUNT(*)`** — 统计物品记录条数，不再使用 `SUM(quantity)` 累加。
- **各冰箱物品分布同样改为记录条数** — `item_count` 字段统一为 `COUNT(i.id)`。

### 过期推算规则完善

- **有保质期但无生产日期**：以"添加日期 + 保质期÷2（对半折算）"推算过期时间。例如保质期 180 天，则 90 天后开始临期提醒。
- **无生产日期也无保质期**：默认为"添加日期 + 5 天"作为过期时间。
- 两种情况均纳入临期判断和汇总看板的预警统计。

### 库存浏览默认展示

- 未选择冰箱时，不再显示空白，默认展示最近到期的 30 件物品（按过期时间升序排列，不足 30 件则全部显示）。
- 物品卡片中，只要有推算出的过期时间就显示。

## v1.0.0 (2026-05-23)

### 导航调整

- 汇总看板调整为导航栏首位，作为系统首页。
- 冰箱管理移至导航栏末位。

### 保质期系统重构

- 物品表单中原"过期日期"字段改为"生产日期"（更贴合实际使用场景：信息可直接从商品包装获取）。
- 新增"保质期"字段，默认以天为单位，支持切换为月或年。
- 过期时间由系统根据"生产日期 + 保质期"自动推算，不再直接展示过期日期输入框。
- 生产日期和保质期均未填写的物品，存放超过 3 天将在汇总看板中预警提醒。

### 数据库变更

- `items` 表 `expire_date` 字段重命名为 `production_date`。
- 新增 `shelf_life_value`（INT）和 `shelf_life_unit`（ENUM: day/month/year）列。

- 预置 8 个物品分类（蔬菜/肉类/水果/乳制品/饮料/调料/速食/其他）。

- 单页面应用，响应式布局。
